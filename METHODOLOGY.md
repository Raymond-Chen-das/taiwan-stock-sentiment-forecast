# 方法論說明

## 研究問題

將 Liu et al.（2025, *Knowledge-Based Systems*）提出的社群媒體情緒空間模型複製並應用於台灣加權股價指數（TAIEX），同時提出以 LLM 語義分類增強看漲指數（BI）的方案，解決 PTT 反諷文化造成的噪音問題。

---

## 資料來源

| 指標 | 來源 | 衡量面向 |
|------|------|----------|
| 關注度指數（AI） | Google Trends | 投資者搜尋行為熱度 |
| 看漲指數（BI） | PTT Stock 板 | 散戶情緒方向 |
| 傳播指數（PI） | PTT Stock 板 | 社群互動強度（PCA 降維） |

- **訓練期**：2021-01-04 – 2024-12-31（970 個交易日）
- **測試期**：2025-01-02 – 2026-01-30（262 個交易日）

### 為什麼排除 YouTube

評估初期曾將 YouTube 納入 PI 資料來源的候選，最終因資料過於稀疏而排除：

| 指標 | PTT | YouTube |
|------|-----|---------|
| 交易日覆蓋率 | 100% | 42.8% |
| 日均數量 | 78.4 篇 | 2.17 支 |
| 2021 年月均 | 1,600+ 篇 | 5–9 支 |

每日中位數僅 1 支影片，對單樣本/天的輸入進行 PCA 降維在統計上毫無意義。最終 BI 與 PI 均採用 PTT 資料，與原論文單一平台的設計一致。

---

## 時間對齊

PTT 有週末與假日資料，TAIEX 只有交易日。處理策略：將所有非交易日的 PTT 指標加總，歸入下一個交易日，模擬散戶在週末累積資訊、週一開盤反映的行為。

```
週五資料  →  週五交易日列
週六 + 週日  →  週一交易日列（加總）
```

## 情緒空間模型

將 AI、BI、PI 三個 Z-score 標準化後的指標視為三維空間中的座標點，每個交易日對應一個點。模型尋找**情緒驅動子空間（SDS）**——歷史上多數點屬於同一趨勢方向（漲或跌）的空間區域。

### HDS 邊界（超維橢球）

SDS 的邊界是三維空間中的**軸對齊橢球體**，滿足以下條件的點位於橢球內：

```
(AI/r_AI)² + (BI/r_BI)² + (PI/r_PI)² ≤ 1
```

三個半徑 `(r_AI, r_BI, r_PI)` 由 PSO 在滾動訓練視窗內最佳化求得。

### PSO 最佳化

- 30 個粒子 × 100 次迭代
- 適應函數 = 訓練視窗內的 `覆蓋率 × 準確率`
- 四個子模型：A（AI×BI）、B（AI×PI）、C（BI×PI）、D（AI×BI×PI）

### 多重過濾預測

只有通過以下所有過濾條件時才發出預測信號：

| 過濾條件 | 門檻 | 目的 |
|----------|------|------|
| Support | ≥ 2 筆歷史匹配 | 避免一次性模式 |
| Gini 不純度 | < 0.48 | 要求趨勢方向明確 |
| K 近鄰一致性 | 多數相同方向 | 局部空間一致性驗證 |

## 修正的方法論問題

在複製原論文過程中發現並修正了兩個常見但隱性的問題：

### 1. Z-score 資料洩漏（Data Leakage）

**問題**：用全期資料計算 μ/σ 做標準化，等於把測試期的統計資訊提前洩漏給訓練期。  
**修正**：僅用訓練期資料計算 μ 和 σ，再套用至測試期。

```python
mu = train_df[col].mean()
sigma = train_df[col].std()
df[f"{col}_zscore"] = (df[col] - mu) / sigma
```

### 2. 預測目標時間偏移（Look-Ahead Bias）

**問題**：若目標標籤 `y[t] = 1` 表示「第 t 天上漲」，而特徵也來自第 t 天，等於用當天資訊預測當天結果。  
**修正**：`y = returns.shift(-1)`，用第 t 天的情緒特徵預測第 t+1 天的漲跌。

## LLM 增強方案

PTT 的推噓機制會因社群文化產生語義噪音：

- 「推 早安大爆崩」— 社交問候語，非看跌信號
- 「噓 又漲了」— 反諷式噓聲，實為看漲情緒

**解決方案**：使用 Claude Opus 4.6 對每篇文章的推噓留言串進行語義分類，輸出 +1（看漲）、0（中性）或 -1（看跌），取代原始推噓比。

**混合 BI 設計**（避免重新標注 970 個訓練日）：

- 訓練期：沿用原始推噓比 BI
- 測試期：改用 LLM 分類 BI

PSO 在推噓比 BI 的模式上訓練，推論時輸入 LLM-BI 信號。

### 分類執行方式

- 透過 Claude Code 互動式判讀，而非 API 批次呼叫
- 判讀依據：文章本體語氣 + 推文整體傾向 + 反諷識別
- 僅分類測試期 261 篇（每日互動量最高的 1 篇），訓練期 970 篇沿用推噓比

**為何選擇互動式而非 API**

- 零 API 成本，適合課堂專題
- 反諷個案可即時確認處理邏輯（如「推 早安大爆崩」「噓 又漲了」等需語境判斷的情況）

**為何採混合 BI（訓練推噓比 / 測試 LLM）**

- 逐篇判讀無法規模化到 970 篇訓練日
- 此設計把 LLM 定位為「測試期信號增強」，PSO 仍在一致的推噓比模式上訓練
- 代價：訓練與測試期的 BI 定義不完全對稱，這是此設計的已知限制

## Grid Search 結果（96 組參數組合）

搜尋範圍：α ∈ {0.15, 0.20, 0.25, 0.30}、θ ∈ {0.60, 0.65, 0.70, 0.75}、視窗 ∈ {20, 40, 60} 天、版本 ∈ {Baseline, LLM-BI}（4 × 4 × 3 × 2 = 96 組）

| 配置 | 版本 | 覆蓋率 | 準確率 |
|------|------|--------|--------|
| w=40, α=0.25, θ=0.75 | **LLM-BI** | 6.2% | **81.2%** |
| w=40, α=0.25, θ=0.75 | Baseline | 6.1% | 68.8% |
| w=20, α=0.20, θ=0.65 | **LLM-BI** | **30.4%** | **62.0%** |
| w=20, α=0.20, θ=0.65 | Baseline | 27.2% | 59.2% |

在 48 組配對（相同 window/α/θ）中，LLM-BI 的覆蓋率多數較佳（33/48），準確率則互有勝負、整體略佔優（26 勝 1 平 21 負）；最一致的改善出現在預設參數（α=0.20, θ=0.65）的三種視窗（+2.9 / +8.6 / +14.2%）與挑選的最佳配置。LLM-BI 落後的配置多落在 α=0.15 與高 θ／長視窗的小樣本區。

## 研究限制與誠實聲明

本研究為課程專題，以下限制與其結果同等重要：

### 1. 小樣本與信賴區間
最高準確率配置（w=40, α=0.25, θ=0.75）的覆蓋率僅 6.2%——262 個測試交易日中僅約 **16 次**發出預測，81.2% 約等於 13/16。此準確率的信賴區間極寬，不應視為穩定表現。**較可信的指標是 62.0% @ 30.4% 覆蓋率**（約 80 次預測）。報告同時列出兩者即為此故。

### 2. Grid Search 的挑選偏誤
96 組參數中「挑出最佳值」本身是一種樂觀偏誤（multiple comparisons）。本研究未保留獨立的驗證組做巢狀驗證，因此最佳組合的表現可能無法外推到未見參數設定。較嚴謹的做法是巢狀交叉驗證或預先註冊參數。

### 3. 混合 BI 的定義不對稱
訓練期使用推噓比 BI、測試期使用 LLM 分類 BI（見上節）。PSO 在推噓比模式上訓練、推論時吃 LLM 信號，兩者並非同一量尺。這是「避免重標 970 個訓練日」的工程折衷，但也意味著訓練/測試的特徵分布不完全一致。

### 4. LLM 分類的非盲性
測試期文章的情緒分類以互動方式判讀，依據為「文章語氣 + 留言整體傾向 + 反諷識別」，**未參考隔日漲跌**。但互動式判讀缺乏可完全稽核的紀錄，仍存在殘餘主觀性；嚴謹版本應改為固定 prompt 的批次 API 分類並保留逐篇輸出以利重現。

### 5. 單一市場、單一測試期
僅涵蓋 TAIEX 與 2025/01–2026/01 一段測試期，且該期間屬特定市場情境。泛化性需以跨市場、跨多段時期的滾動測試進一步驗證。

### 6. 程式面註記
- 指標 builder（`processors/`）會輸出全期 `*_zscore` 欄位，雖然模型刻意改用 `*_raw` 並於訓練期重算 z-score（見 `src/utils/preprocessing.py`），該全期欄位仍是潛在的洩漏陷阱，未來可移除。
